# 広告とプライバシー

> 親ドキュメント：[../CLAUDE.md](../CLAUDE.md)

## 本アプリの位置づけ

本アプリは **保育園入園を検討する保護者向けのユーティリティアプリ**であり、子ども自身が利用するアプリではない。
そのため：

- **対象年齢：一般（成人＝保護者）**
- **Google Play Families ポリシー：対象外**
- **AdMob「子ども向けコンテンツ扱い」フラグ：OFF**
- **配信地域：日本のみ**（UMP SDK / GDPR 同意フローは導入しない）

将来海外配信する場合は UMP SDK と地域別同意ロジックを追加する。

-----

## AdMob 広告

### バナー広告

- 結果画面の下部に常時表示
- 結果表示エリアと十分な余白を設けて誤クリック防止（最低 16dp）

### インタースティシャル広告

- 結果表示画面への遷移直前に表示
- **頻度キャップ：同一セッション内では1回のみ。アプリのバックグラウンド復帰でセッションリセット**
  - 「セッション」= アプリを起動してから完全にバックグラウンドへ移行するまでの期間
  - 同一セッション内で自治体を切り替えて再計算しても、2回目以降は表示しない
  - これによりユーザー体験を損なわず、機会収益も確保

### AdMob 管理画面／SDK の設定

- **広告コンテンツのフィルタリング：既定（PG 相当）で十分**
  - G レーティングへの強制は保護者向けアプリでは eCPM を下げるのみで、ポリシー上も必要ない
- **子ども向けコンテンツ扱い：OFF**（対象は成人保護者のため）
- パーソナライズ広告：日本のみ配信のため通常通り（GDPR-K考慮不要）

### SDK 側での明示設定

Google Mobile Ads SDK の `RequestConfiguration` で **「子ども向けではない」ことを明示**する。
未指定のままだとポリシー警告の対象になる可能性があるため必須。

```dart
await MobileAds.instance.updateRequestConfiguration(
  RequestConfiguration(
    tagForChildDirectedTreatment: TagForChildDirectedTreatment.no,
    tagForUnderAgeOfConsent: TagForUnderAgeOfConsent.no,
    maxAdContentRating: MaxAdContentRating.pg,
  ),
);
```

## ローカル保存ポリシー

ユーザーが入力した家庭情報は **要配慮個人情報** を含む（障害等級、生活保護受給、就労状況など）。

- `shared_preferences`（平文XML）には保存しない
- **`flutter_secure_storage 10.x`** を使用する
- 直近の入力値は永続保存し、起動時に再表示する
- 設定画面に「保存データを削除」ボタンを必ず提供する

### 初期化コード例（必須オプション）

`flutter_secure_storage 10.x` では `encryptedSharedPreferences: true` の明示が必須。

```dart
const storage = FlutterSecureStorage(
  aOptions: AndroidOptions(
    encryptedSharedPreferences: true,
  ),
);
```

> ⚠️ 軽微な UI 設定（最後に選んだ自治体ID、テーマ等）に限り `shared_preferences` を併用することは可。

### 起動時の初期化

`flutter_secure_storage` は Android Keystore を使用するため、初回読み込みに数100ms〜数秒かかる場合がある。
スプラッシュ画面または非同期 `FutureProvider` を用いてバックグラウンドで初期化し、
UI ブロックを避ける。

## 外部送信

- 入力された家庭情報は **外部サーバーへ送信しない**
- AdMob による広告識別子（広告ID／端末識別子）は Google に送信される
  - これをプライバシーポリシーおよびデータセーフティで明示する

-----

## Google Play 公開時に必要なもの

### プライバシーポリシー

- URL を Google Play Console に登録（GitHub Pages 等で公開）
- 内容に最低限含めるもの：
  - 収集する情報の種類（広告ID／端末識別子）
  - アプリ内で扱う要配慮個人情報の取り扱い（端末内のみ保存、暗号化）
  - 第三者提供（Google AdMob）
  - 削除リクエストの手段（アプリ内「保存データを削除」）
  - 問い合わせ先

### データセーフティセクション（Google Play Console）

| カテゴリ                   | 内容                                              |
|------------------------|-------------------------------------------------|
| アプリ内で収集（端末外送信なし）       | 健康情報相当：障害等級・要介護度／個人情報：就労状況・家族構成・生活保護受給有無    |
| 第三者と共有                | 広告ID／端末識別子（Google AdMob 経由）                    |
| 暗号化転送                  | はい                                              |
| データ削除リクエスト方法           | アプリ内「保存データを削除」ボタン                              |
| 子ども向け                  | いいえ（保護者向けユーティリティ）                              |

### その他

- 対象年齢設定：**一般（13歳以上）**
- 配信地域：**日本のみ**
- アプリアイコン（512×512）、フィーチャーグラフィック（1024×500）、スクリーンショット最低2枚

-----

## バージョニングと年度更新

- 指数ルールは毎年度更新されるため、`pubspec.yaml` の `version` を年度切替時に必ず上げる
- 自治体選択画面で各自治体の対応年度を表示（自治体ごとに更新タイミングが異なるため）
- 年度更新の際は `ScoringRule.fiscalYear` を更新し、ユニットテストのゴールデンケースも更新
