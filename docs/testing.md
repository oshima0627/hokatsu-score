# テスト方針

> 親ドキュメント：[../CLAUDE.md](../CLAUDE.md)

## スコア計算ユニットテスト

スコア計算は副作用がなく純粋関数で構成されるため、`test/scoring/<municipality>_test.dart` で網羅テスト必須。

### 各自治体ファイルが最低限カバーすべきケース

#### 月の就労時間境界値（那覇市の例）

すべての境界の両側を確認する：

| 入力(h) | 期待点数 | 意味        |
|-------|------|-----------|
| 63    | 0    | 64h未満で0点  |
| 64    | 15   | 64h以上で15点 |
| 89    | 15   | 90h未満は15点 |
| 90    | 19   | 90h以上で19点 |
| 119   | 19   | 120h未満は19点 |
| 120   | 22   | 120h以上で22点 |
| 139   | 22   | 140h未満は22点 |
| 140   | 26   | 140h以上で26点 |
| 159   | 26   | 160h未満は26点 |
| 160   | 30   | 160h以上で30点 |

#### 自営業（証明書なし）の上限キャップ（那覇市の例）

`selfEmployedNoProof` は就労時間段階の点数を**上限9点**で頭打ちにする：

| 入力(h) | employed の点数 | selfEmployedNoProof の点数 | 意味                    |
|-------|-------------|----------------------|-----------------------|
| 63    | 0           | 0                    | 64h未満は元が0なのでキャップ関係なし |
| 64    | 15          | 9                    | 15→9にキャップ             |
| 90    | 19          | 9                    | 19→9にキャップ             |
| 160   | 30          | 9                    | 30→9にキャップ             |

#### 排他ロジック

- ひとり親世帯 + ひとり親みなしが両方true → より高い方の点数のみ加算

#### 大きな減点

- `acceptsLeaveExtension = true` で実質マイナスになること（那覇市は−500）

#### 初期値

- すべての `bool` フラグが `false`、`workStatus = notSpecified`、両親とも `monthlyWorkHours = 0`、`disabilityGrade = none`、`careLevel = none` で**合計0点**

### ゴールデンテスト

各自治体の公式PDFに掲載されている計算例の数値を再現するテストを **1自治体あたり最低1ケース** 用意する。

## ウィジェットテスト（推奨）

- 自治体未選択で次画面に進めないこと
- 結果画面に内訳と免責文言が表示されていること
- シェアボタンをタップすると `share_plus` の API が呼ばれること（モック）

-----

## 実行コマンド

```bash
flutter test
flutter analyze
```

CI 導入時はこの2つを必須チェックとする。
